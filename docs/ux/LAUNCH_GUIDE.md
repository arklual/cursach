## Что реализовано

### ✅ Созданные компоненты и сервисы

1. **WorkflowValidatorService** (`services/workflow-validator.service.ts`)
   - Валидация графа перед запуском
   - Обнаружение циклов, проверка подключений нод
   - Индикатор статуса: 🟢 Готов / 🟡 Требует внимания / 🔴 Ошибка

2. **CanvasEmptyComponent** (`components/canvas-empty/canvas-empty.component.ts`)
   - Пустое состояние холста с быстрым добавлением нод
   - 4 популярные ноды: Trigger, A/B Fork, HTTP, Code
   - Клик по кнопке → нода добавляется в центр холста

3. **StatisticsTermsService** (`services/statistics-terms.service.ts`)
   - Словарь терминов с пояснениями
   - 2 режима: Базовый ("Конверсия") / Эксперт ("p̂")
   - Формулы, интерпретации, рекомендации

4. **OnboardingTourComponent** (`components/onboarding-tour/onboarding-tour.component.ts`)
   - 6 шагов: Приветствие → Палитра → Холст → Инспектор → Запуск → Результаты
   - Сохранение прогресса в localStorage
   - Индикатор прогресса (точки)

5. **Simulation Scenarios** (обновлён `services/simulation.service.ts`)
   - 3 реалистичных сценария: A/B тест кнопки, Воронка онбординга, Тест оплаты
   - Осмысленные данные вместо случайных чисел
   - Контекст для каждого сценария

## 🧪 Тестирование

### 1. Быстрая проверка (5 минут)

```bash
cd frontend
npm start
```

**Шаги:**
1. Откройте http://localhost:4200
2. Перейдите к списку workflow
3. Создайте новый workflow
4. **Проверьте пустое состояние холста**:
   - Видите ли 4 кнопки (Trigger, A/B Fork, HTTP, Code)?
   - Добавляется ли нода при клике?
5. **Проверьте валидацию**:
   - Добавьте Trigger → индикатор 🟢
   - Добавьте A/B Fork без подключения → индикатор 🟡
   - Подключите Trigger → A/B → индикатор 🟢
6. **Проверьте онбординг**:
   - Откройте в инкогнито (или сбросьте localStorage)
   - Должен появиться тур из 6 шагов

### 2. Проверка валидации графа

**Сценарий A: Ошибка (🔴)**
```
1. Создайте workflow
2. Добавьте A/B Fork (без Trigger)
3. Ожидание: 🔴 "Ошибка: Нет стартовой ноды"
```

**Сценарий B: Предупреждение (🟡)**
```
1. Добавьте Trigger
2. Добавьте HTTP, но не подключайте
3. Ожидание: 🟡 "Требует внимания: Нода "HTTP" не подключена"
```

**Сценарий C: Готов (🟢)**
```
1. Trigger → HTTP → Join
2. Все ноды подключены
3. Ожидание: 🟢 "Готов к запуску"
```

### 3. Проверка симуляции с новыми данными

```typescript
// В simulation.service.ts теперь есть сценарии:
SIMULATION_SCENARIOS = [
  {
    name: 'A/B тест кнопки',
    baselineConversion: 0.25,  // 25%
    expectedUplift: 0.06       // +6 п.п.
  },
  // ...
]
```

**Шаги:**
1. Создайте workflow: Trigger → A/B Fork → 2x HTTP → Join
2. Нажмите "Симуляция (500)"
3. Проверьте результаты:
   - Конверсии реалистичны (~25-31%)
   - p-value рассчитывается
   - Рекомендация появляется

### 4. Проверка терминов

**Где смотреть:**
- Analytics Modal (двойной клик по ноде)
- A/B Config Modal
- Experiment Results Modal

**Ожидается:**
- В базовом режиме: "Конверсия", "Доверительный интервал", "Значимость"
- В экспертном режиме: "p̂", "CI95%", "p-value"

## 🐛 Известные проблемы и решения

### Проблема 1: Онбординг не появляется
**Решение:**
```javascript
// В консоли браузера:
localStorage.removeItem('fluxpilot_onboarding_completed');
location.reload();
```

### Проблема 2: Валидация не обновляется
**Причина:** effect не срабатывает
**Решение:** Проверьте, что `graphLoaded()` возвращает `true`

### Проблема 3: CanvasEmpty не центрирует ноду
**Решение:** Метод `centerView()` вызывается с задержкой 100ms — увеличьте до 300ms
