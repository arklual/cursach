# Информационная архитектура и пользовательские сценарии

## 🎯 Принципы редизайна

### 1. Разделение режимов работы
**Проблема**: Все функции в одном интерфейсе создают когнитивную нагрузку.

**Решение**: 3 чётких режима:
- **Редактор** — сборка workflow (ноды, связи, настройки)
- **Запуск** — тестирование и симуляция
- **Результаты** — аналитика и метрики

### 2. Прогрессивное раскрытие сложности
**Проблема**: Статистические термины (p̂, CI, Var) пугают новичков.

**Решение**:
- Базовый уровень: "Конверсия", "Успешность", "Разница"
- Продвинутый уровень (по клику): "p̂ = 0.25", "CI95% = [0.21–0.29]"
- Переключатель "Простой/Эксперт" режим

### 3. Визуальная обратная связь
**Проблема**: Непонятно, готов ли workflow к запуску.

**Решение**:
- Индикатор готовности графа (все ноды подключены?)
- Предупреждения о проблемах до запуска
- Анимация прохождения трафика по нодам

### 4. Осмысленные заглушки вместо "моков"
**Проблема**: "Simulate 500 users" — абстрактное действие.

**Решение**:
- "Тестовый запуск" — 1 запрос, проверка логики
- "Симуляция трафика" — 100/500/1000 пользователей для метрик
- Чёткое объяснение: "Зачем это нужно"

## 🚀 Пользовательские сценарии

### Сценарий 1: Быстрый старт (5 минут)
**Цель**: Собрать первый workflow и запустить

1. **Открывает главный экран** → видит список workflow
2. **Нажимает "+ Новый"** → выбирает шаблон "A/B тест"
3. **Редактор**: 
   - Видит 2 готовые ноды (A и B), соединённые линией
   - Меняет текст на кнопках в настройках
4. **Нажимает "Тест-запуск"** → видит логи выполнения
5. **Нажимает "Симуляция (100)"** → видит конверсию
6. **Делает скриншот** → готово для отчёта

**Успех**: Workflow запущен, результаты видны за ≤5 минут

---

### Сценарий 2: Исправление ошибок
**Цель**: Понять, почему workflow не работает

1. **Нажимает "Запустить"** → видит ошибку "Нода 2 не подключена"
2. **Красная рамка** вокруг проблемной ноды
3. **Подсказка**: "Соедините выход ноды 1 с входом ноды 2"
4. **Перетаскивает связь** → ошибка исчезает
5. **Повторный запуск** → успешно

**Успех**: Пользователь понимает проблему и может её исправить

---

### Сценарий 3: Анализ результатов
**Цель**: Понять, какой вариант лучше

1. **После симуляции** → видит "Вариант B: +6 п.п."
2. **Нажимает "📊 Подробнее"** → открывается аналитика
3. **Видит графики**:
   - Воронка: сколько дошло до каждой ноды
   - Конверсия по времени: растёт ли стабильно
4. **Читает рекомендацию**: "B лучше (p < 0.05)"
5. **Нажимает "Экспорт"** → скачивает JSON с данными

**Успех**: Пользователь понимает результат и может его интерпретировать

## 📊 Упрощение терминов

| Старый термин | Новый термин | Пояснение (по клику) |
|--------------|--------------|---------------------|
| p̂ | Конверсия | Доля пользователей, выполнивших целевое действие |
| CI95% | Доверительный интервал | "Истинная конверсия с вероятностью 95% находится в этом диапазоне" |
| p-value | Значимость | "Вероятность, что разница случайна. < 0.05 — разница реальна" |
| Var(p̂) | Дисперсия | "Разброс значений конверсии" |
| Traffic allocation | Распределение трафика | "Сколько процентов пользователей увидят каждый вариант" |
| Randomization mode | Метод распределения | "Как именно определять, кто в какой вариант попадёт" |
| Stopping rules | Правила остановки | "Когда прекращать тест: после N пользователей или при достижении значимости" |

## 🎨 Визуальная иерархия

### Приоритеты элементов (от важного к второстепенному)

1. **Кнопка "Запустить"** — самая яркая, справа вверху
2. **Холст** — центральная область, 60% экрана
3. **Индикатор готовности** — рядом с кнопкой запуска
4. **Палитра нод** — слева, 20% экрана
5. **Настройки ноды** — справа, 20% экрана
6. **Логи** — внизу, сворачиваемая панель

### Цветовая схема состояний

| Состояние | Цвет | Использование |
|-----------|------|---------------|
| Успех | 🟢 #22c55e | Завершено, конверсия, значимость |
| Ошибка | 🔴 #ef4444 | Проблема, нет связи, p > 0.05 |
| Предупреждение | 🟡 #f59e0b | Мало данных, низкая мощность |
| Информация | 🔵 #3b82f6 | Подсказки, детали |
| Нейтрально | ⚪ #64748b | Неактивные элементы |
