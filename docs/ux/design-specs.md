# Спецификации дизайна компонентов

## 1. Главный экран (Workflow List)

### Назначение
Точка входа в приложение. Позволяет выбрать существующий workflow или создать новый.

### Структура
```
┌────────────────────────────────────────────────────────────┐
│ Header                                                      │
├────────────────────────────────────────────────────────────┤
│ Hero-секция с названием и быстрым созданием                │
├────────────────────────────────────────────────────────────┤
│ Секция "Мои workflow" (карточки)                           │
├────────────────────────────────────────────────────────────┤
│ Секция "Шаблоны" (преимущества)                            │
├────────────────────────────────────────────────────────────┤
│ Footer с документацией                                      │
└────────────────────────────────────────────────────────────┘
```

### Компоненты

#### Header
- **Логотип**: "Δ FluxPilot" (слева)
- **Навигация**: "Мои workflow", "Шаблоны", "Документация"
- **Профиль**: Аватар + имя (справа)

#### Hero-секция
- **Заголовок**: "Workflow-редактор для A/B-тестов"
- **Подзаголовок**: "Визуально создавайте пайплайны, тестируйте гипотезы и анализируйте результаты"
- **Кнопка**: "+ Новый workflow" (большая, акцентная)

#### Карточка workflow
```
┌─────────────────────────┐
│ 📊 Тест кнопки          │
│                         │
│ 3 ноды • 2 связи        │
│                         │
│ 🟢 Активен              │
│ Обновлено: 2 часа назад │
│                         │
│ [▶ Запустить] [✏️]     │
└─────────────────────────┘
```

**Состояния карточки:**
- 🟢 Активен — workflow запущен
- 🟡 Черновик — не был запущен
- 🔴 Ошибка — проблема при выполнении

#### Шаблоны
Готовые конфигурации для быстрого старта:

| Шаблон | Описание | Ноды |
|--------|----------|------|
| A/B тест | Сравнение 2 вариантов | Trigger → A/B Fork → 2x HTTP → Join |
| HTTP пайплайн | Последовательные запросы | Trigger → HTTP → HTTP → HTTP |
| Фильтрация | Отбор по условиям | Trigger → Filter → Code |
| Парсинг | Сбор данных с сайтов | Trigger → HTTP → Code → Save |

## 2. Редактор workflow

### Назначение
Основной интерфейс для создания и редактирования пайплайнов.

### Layout (CSS Grid)
```css
.editor-container {
  display: grid;
  grid-template-rows: 64px 1fr 200px; /* header, main, logs */
  grid-template-columns: 280px 1fr 320px; /* palette, canvas, inspector */
  height: 100vh;
}
```

### Header редактора
```
┌─────────────────────────────────────────────────────────────┐
│ ← Назад  │  [Название] ✏️  │  🟢 Готов  │  [💾] [▶ Запуск] │
└─────────────────────────────────────────────────────────────┘
```

**Элементы:**
- **Кнопка "Назад"**: Возврат к списку workflow
- **Название**: Редактируемое поле (клик → edit)
- **Индикатор готовности**:
  - 🟢 "Готов к запуску" — все ноды подключены
  - 🟡 "Требует внимания" — есть предупреждения
  - 🔴 "Ошибка" — критические проблемы
- **Кнопка "Сохранить"**: Ctrl+S, автосохранение каждые 30 сек
- **Кнопка "Запуск"**: Выпадающее меню:
  - "Тест-запуск (1 запрос)"
  - "Симуляция (100 пользователей)"
  - "Симуляция (500 пользователей)"
  - "Симуляция (1000 пользователей)"

### Палитра нод (левая панель)
```
┌──────────────────────────┐
│ 🔍 Поиск нод...          │
├──────────────────────────┤
│ 📥 Вход                  │
│   [🎯 Trigger]           │
│   [🌐 HTTP]              │
├──────────────────────────┤
│ 🔀 Логика                │
│   [⚖ A/B Fork]           │
│   [📊 Filter]            │
│   [🔁 Loop]              │
├──────────────────────────┤
│ ⚡ Действия              │
│   [💻 Code]              │
│   [📤 Webhook]           │
├──────────────────────────┤
│ 📤 Выход                 │
│   [⏹ Join]               │
│   [💾 Save]              │
└──────────────────────────┘
```

**Взаимодействие:**
- Drag & drop на холст
- Клик → информация о ноде (тултип)
- Поиск фильтрует по названию и описанию

**Категории нод:**
1. **Вход** — начало пайплайна
2. **Логика** — ветвление и условия
3. **Действия** — HTTP-запросы, код
4. **Выход** — завершение, сохранение

```

**Функции:**
- **Pan**: Перетаскивание правой кнопкой / пробел + drag
- **Zoom**: Ctrl/Cmd + колесо мыши (30%–200%)

- **Создание связи**: Drag от точки-коннектора
- **Удаление связи**: Клик по линии → Delete
- **Мультиселект**: Shift + клик / рамка

**Пустое состояние:**
```

## 5. Состояния и обратная связь

### Индикатор готовности графа
Проверка перед запуском:

**🟢 Готов к запуску:**
- Есть хотя бы 1 нода
- Все ноды подключены (нет "висячих")
- Есть стартовая нода (Trigger)
- Есть конечная нода (опционально)

**🟡 Требует внимания:**
- Нет конечной ноды (предупреждение)
- Подозрительно мало нод (< 2)
- Не настроены параметры ноды

**🔴 Ошибка:**
- Нет стартовой ноды
- Есть ноды без входящих связей (кроме Trigger)
- Есть ноды без исходящих связей (кроме Exit)
- Циклические зависимости

### Анимации и переходы

| Событие | Анимация | Длительность |
|---------|----------|--------------|
| Добавление ноды | Fade in + scale up | 200ms |
| Удаление ноды | Fade out + scale down | 150ms |
| Создание связи | Draw line from source to target | 300ms |
| Запуск workflow | Pulse on active nodes | 500ms loop |
| Успех | Green flash | 300ms |
| Ошибка | Red shake | 400ms |

### Тултипсы и подсказки

**Контекстные подсказки:**
- При наведении на ноду: краткое описание
- При первом открытии редактора: тур по интерфейсу
- При ошибке: что не так + как исправить

**Формат тултипса:**
```
┌────────────────────────────┐
│ 🌐 HTTP Request            │
│                            │
│ Отправляет HTTP-запрос     │
│ к указанному URL.          │
│                            │
│ Поддерживает: GET, POST,   │
│ PUT, DELETE                │
│                            │
│ [📖 Документация]          │
└────────────────────────────┘
```

## 6. Адаптивность

### Брейкпоинты

| Размер | Layout | Особенности |
|--------|--------|-------------|
| ≥1440px | 3 колонки (280px, 1fr, 320px) | Полный интерфейс |
| 1024–1439px | 2 колонки (палитра сворачиваемая) | Инспектор поверх |

- Просмотр списка workflow

## 7. Доступность (WCAG 2.1 AA)

### Требования
- **Контрастность текста**: ≥ 4.5:1
- **Контрастность UI элементов**: ≥ 3:1
- **Минимальный размер кликабельной области**: 44×44px
- **Фокус клавиатуры**: Видимый outline
- **ARIA-метки**: Для всех интерактивных элементов
- **Поддержка скринридеров**: Семантическая разметка

### Клавиатурная навигация
| Клавиша | Действие |
|---------|----------|
| Tab / Shift+Tab | Переключение фокуса |
| Enter / Space | Активация кнопки |
| Escape | Закрыть модальное окно |
| Delete / Backspace | Удалить выделенное |
| Ctrl+C / Ctrl+V | Копировать / вставить ноду |
| Ctrl+Z | Отменить действие |
| Стрелки | Перемещение ноды (при выделении) |
| Ctrl+Scroll | Зум холста |
| Пробел + Drag | Pan холста |
