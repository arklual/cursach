> Цель — построить собственный движок исполнения workflow с отказоустойчивостью и масштабированием.

n8n — однопоточный runtime на Node.js. Поэтому можно сделать улучшенную версию — **Java-движок исполнения графа**, где:´

- Distributed tasks для узлов, балансировка.
- State recovery: чекпоинты/снапшоты, детерминированный replay.
- Pause/resume/cancel, ретраи идемпотентные.
- Sandbox узлов (литкод)
- Debugger: брейкпоинты на узлах, пошаговое исполнение, просмотр входов/выходов.
- Саги-паттерн
- Версионирование и миграции (git)
- Импорт/экспорт workflow в/из JSON.
- фича флаги и A/B тесты?

**Т. е. если кратко описать проект:**

Промышленный Java-движок исполнения графов с отказоустойчивостью и горизонтальным масштабированием, совместимый по концепции с n8n (узлы/связи), но с версионированем, упрощённым debug-ом, snapshot-ами, фича-флагами.

**Важно (имхо):** более чётко определить сегмент — не распыляться как n8n на все задачи, а, например, сделать больше упор на продуктовые исследования или какую-то другую сферу. Потому что мы не сможем покрыть такое количество интеграций, сколько уже реализовано в n8n, поэтому логично, как мне кажется, сделать интеграции для какой-то одной сферы и заточить UI/UX для них.

**Функциональные требования:**

- Создание/редактирование воркфлоу (ноды/ребра)
- Возможность параллельного исполнения нод
- Возможность масштабирование нод на несколько машин; автоматическая балансировка воркфлоу на машины
- Возможность отслеживать изменения в воркфлоу, откатывать предыдущие версии
- low priority - Возможность ставить брейкпоинты
- полностью просматривать стейт
- low priority - Возможность ставить исполнение на паузу и продолжать с этого же стейта (сохранять снэпшот)
- Поддержка: HTTP, DB Query (postgres, redis), Delay/Timer, иф, потоки данных, Code Node с песочницей: js, python, sql, интеграции (AI, tg, …)
- -low priority - gRPC
- Триггеры воркфлоу: ручной запуск, scheduler, вубхука
- low-priority - триггер интеграции (тг-бот, …)
- Возможность экспортировать воркфлоу в json
- Возможность импортировать воркфлоу из json
- Поднимать бд прям из интерфейса
- -low low priority: Создание собственных интеграций
